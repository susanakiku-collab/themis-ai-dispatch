🚐 概要

システムの役割

  • キャストと車両をまとめて管理します。
  • 予定表と実際の送りを分けて扱えます。
  • 当日運用では、自動配車と便別試算ができます。
  • 結果はコピーやLINE送信でドライバー共有できます。

ホーム画面で見えるもの

  • 本日の人数、車両台数、Plan件数、Actual件数
  • 完了件数、キャンセル件数
  • 月間の車両別集計
  • 主要画面へのクイック移動

🧭 画面別ガイド

画面 主な用途 現在のポイント
ホーム 当日の全体状況と月間集計の確認 今日の件数確認、当日運用 / 予定表 / 実際の送りへすぐ移動できます。
キャスト管理 キャストの登録、更新、検索、CSV入出力 住所入力、座標貼り付け、方面判定、ルート確認に対応しています。
車両 車両マスタ管理と走行実績エクスポート 担当方面、帰宅方面、LINE ID、状態、ドライバー名を管理できます。

予定表 送迎予定の登録、CSV入出力 時間帯 × 方面一覧でまとめて確認できます。

設定 ログイン情報、起点確認、全体入出力、リセット、履歴 全体エクスポート / インポート、全消去系ボタン、履歴確認ができます。

キャスト管理で使う項目

  • 氏名、想定距離、住所、方面、メモ、座標、電話
  • 住所入力後に Enter で座標取得できます。
  • 「住所から方面判定」で方面の自動反映ができます。
  • 「ルート」で地図確認ができます。
  • CSVのインポート / エクスポートに対応しています。

車両画面で使う項目

  • 車両ID、担当方面、帰宅方面、帰宅座標、乗車可能人員
  • ドライバー名、LINE ID、状態、メモ
  • 車両CSVのインポート / エクスポートに対応しています。
  • 走行実績は期間指定で一覧表示とCSV出力ができます。

当日運用で使う機能

  • 自動配車
  • 確定（月間へ反映）
  • Actual全消去
  • 配車可能車両の全選択 / 全解除
  • ラスト便車両解除
  • 対象便ごとの試算と試験的な自動配車
  • 結果コピー、LINE送信、日次走行距離保存

予定表 / 実際の送りの違い

  • 予定表 は事前計画用です。
  • 実際の送り は当日の実績管理用です。
  • 実際の送りには「予定から追加」があります。
  • 完了やキャンセルは実際の送り側で管理します。

🛠️ 基本運用の流れ

  1. キャスト管理 で送り先情報を登録します。
  2. 車両 で配車に使う車両情報を登録します。
  3. 予定表 で当日の予定を作成します。必要に応じてCSVも使えます。
  4. 当日運用 で日付を合わせ、使用車両を選び、自動配車または試算を行います。
  5. 実際の送り で完了 / キャンセルなどの状態を反映します。
  6. 当日運用 に戻って日次走行距離を保存し、必要なら「確定（月間へ反映）」を実行します。
  7. ホーム で本日状況と月間集計を確認します。
予定表、当日運用、実際の送りは日付が連動する前提です。日付を切り替えたら、対象日が合っているか確認してから操作してください。

⚙️ 現在の仕様

住所・座標まわり

  • 住所入力から座標取得できます。
  • 同一住所はジオコードキャッシュを利用します。
  • 距離計算用のキャッシュも使います。
  • 距離は起点 THEMIS 基準で計算する運用です。
  • 地図確認にはルートボタンや配車結果マップを使えます。

データ・入出力

  • キャストCSV、車両CSV、予定CSVに対応しています。
  • 設定画面から全体エクスポート / インポートができます。
  • 履歴欄に主要操作のログを残します。
  • 月間集計は日次走行距離の保存内容をもとに反映します。

配車・状態管理

  • 当日運用では対象便ごとの試算ができます。
  • 自動配車結果はコピーとLINE送信ができます。
  • ラスト便車両の手動指定と解除に対応しています。
  • 実際の送りの状態は pending / done / cancel ベースで管理します。
  • 完了・キャンセル済みは、その後の対象から外れる前提で画面が連動します。

一覧表示

  • 予定表と実際の送りは「時間帯 × 方面 一覧」で確認できます。
  • 一覧の色分けは未完了、完了、キャンセルに対応しています。
  • 車両画面ではドライバー別・日別の走行実績エクスポートができます。
このマニュアルは、現在の実装に合わせた運用説明です。配車ロジックの内部スコアや細かな優先度は今後の改修で変わる可能性がありますが、画面操作としてはこの手順で問題ありません。

💡 運用のコツ

入力時

  • 住所と座標は、できるだけ正確に登録してください。
  • 方面が自動判定と違う場合は、必要に応じて手修正してください。
  • 車両の LINE ID は通知運用に合わせて揃えると管理しやすくなります。

当日運用時

  • 先に予定表を整えておくと、当日の「予定から追加」が使いやすくなります。
  • 配車前に使用車両のチェック状態を確認してください。
  • 業務終了時は日次走行距離を保存しておくと、月間集計が揃います。
